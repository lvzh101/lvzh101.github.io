Hexo常用命令总结

Hexo 是一个快速、简洁且高效的博客框架，基于 Node.js 实现。它能将 Markdown 文件极速渲染成静态网页，并且搭配丰富的主题和插件，是开发者和写作爱好者的首选工具之一。

在使用 Hexo 进行博客的日常写作与维护时，有几个常用命令可以说是必须掌握的。下面为大家整理了一份使用 Hexo 构建博客时的常用命令备忘录。

默认情况下，访问网址为：http://localhost:4000/。你只要按 Ctrl + C 就能关闭这个服务。

提示：目前较新的 Hexo 版本中，hexo s 支持“热更新”，也就是说在启动服务期间你修改 Markdown 保存后，直接在浏览器刷新就能看到效果，无需重新执行 hexo g。

5. 清除缓存

如果你修改了配置文件（如 _config.yml）或者更换了主题，发现重新渲染后页面并没有生效，通常是因为之前生成的 .html 文件或者缓存作祟。这时使用 hexo clean 清除本地的数据库和生成的静态文件（也就是删掉 public 文件夹）通常都能“包治百病”。

上面这些命令基本能覆盖建立博客后 99% 的核心使用场景了。只要熟练使用：hexo n (写文章) -> hexo clean && hexo g (编译) -> hexo d (部署发布) 的操作流，你就能把更多的精力集中在输出文字本身，这也是静态博客如此吸引人的魅力所在！
