四表五链

iptables 的“四表五链”是对 Linux netfilter 防火墙体系的经典概括：

  • 四表：raw、mangle、nat、filter
  • 五链：PREROUTING、INPUT、FORWARD、OUTPUT、POSTROUTING

可以理解成：

  • 表（table）：做哪一类处理
  • 链（chain）：在数据包流转的哪个时机处理

一、四表是什么

1. filter 表

这是最常用的表，主要负责过滤数据包。

常见动作：

  • 放行：ACCEPT
  • 丢弃：DROP
  • 拒绝：REJECT

典型用途：

  • 允许 22 端口 SSH
  • 禁止某个 IP 访问
  • 限制某类流量进入本机

2. nat 表

负责网络地址转换。

常见用途：

  • SNAT：源地址转换
  • DNAT：目标地址转换
  • MASQUERADE：动态伪装，常用于家庭路由器共享上网
  • 端口映射

典型场景：

  • 外网访问内网服务器
  • 多台内网机器共享一个公网 IP 上网

回包为什么还能回到真正的内网机器？
因为 NAT 网关自己记了映射关系，不是接收方知道的。

比如网关会维护一张表，大概像这样：

192.168.1.100:50000 ↔ 1.2.3.4:62001
于是流程就是：

  1. 内网主机发请求
    192.168.1.100:50000 -> 8.8.8.8:80

  2. NAT 网关改写后发出去
    1.2.3.4:62001 -> 8.8.8.8:80

  3. 接收方看到的源
    接收方只看到：

1.2.3.4:62001

所以它回包时发给：

8.8.8.8:80 -> 1.2.3.4:62001

  1. NAT 网关收到回包
    网关查自己记录的映射表，发现：

1.2.3.4:62001 对应的是 192.168.1.100:50000
于是再把目标改回去，转给内网主机：

8.8.8.8:80 -> 192.168.1.100:50000

所以不是接收方知道原始 IP，
而是 NAT 网关靠自己的连接跟踪表，把回包“认领”回去。

一句话总结

  • 源 IP 被 NAT 改写后，接收方看到的就是改写后的 IP
  • 接收方不知道原始源 IP
  • 真正负责“记住原始发送者是谁”的，是中间的 NAT 网关

3. mangle 表

负责修改数据包内容，比如一些报文标记、TOS、TTL 等。

典型用途：

  • 给数据包打 mark
  • 配合策略路由使用
  • 修改某些头部字段

这个表更偏高级玩法，普通防火墙规则里用得没 filter 和 nat 多。

4. raw 表

负责连接跟踪前的特殊处理。

最常见用途：

  • 配置数据包不进入连接跟踪（NOTRACK）

它优先级很高，但日常使用相对少。

二、五链是什么

链表示数据包在内核网络协议栈里流动时，会经过的几个关键位置。

1. PREROUTING

数据包刚进入网卡，还没决定路由前。

适合做：

  • DNAT
  • 提前修改包
  • 某些标记操作

2. INPUT

数据包的目标就是本机。

比如：

  • 别人 SSH 到你的服务器
  • 浏览器访问你机器上的 Web 服务

适合做：

  • 控制哪些流量能进入本机

3. FORWARD

数据包不是发给本机，而是经过本机转发。

比如：

  • 你的 Linux 机器充当路由器
  • 一台机器经过这台服务器转发到另一台

适合做：

  • 控制转发流量是否允许通过

4. OUTPUT

数据包是本机发出去的。

比如：

  • 服务器主动 curl 某网站
  • 本机发起 DNS 请求

适合做：

  • 控制本机程序对外访问

5. POSTROUTING

数据包已经决定了路由，即将发出网卡前。

适合做：

  • SNAT
  • MASQUERADE
